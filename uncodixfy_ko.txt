# Uncodixify

이 문서는 UI를 만들 때 최대한 "Codex스럽지 않게" 행동하도록 가르치기 위해 존재합니다.

Codex UI는 기본적인 AI 미학입니다. 부드러운 그라디언트, 떠다니는 패널, eyebrow 라벨, 설명을 위한 장식 문구, 대시보드형 히어로 섹션, 과하게 둥근 모서리, transform 애니메이션, 극적인 그림자, 억지로 고급스러워 보이려는 레이아웃. 이런 건 전형적으로 "AI가 만들었구나"라는 느낌을 줍니다. 왜냐하면 가장 쉬운 길을 그대로 택한 결과이기 때문입니다.

이 문서는 그런 패턴을 깨기 위한 가이드입니다. 아래에 적힌 것은 Codex UI가 기본적으로 하는 행동들입니다. 당신의 역할은 이런 패턴을 알아보고, 완전히 피하고, 사람이 실제로 설계한 것처럼 기능적이고 정직한 인터페이스를 만드는 것입니다.

이 문서를 읽는다는 건 "무엇을 하지 말아야 하는지" 배우는 것입니다. 금지된 패턴은 빨간불이고, 정상적인 구현 방식은 청사진입니다. 이것들을 엄격하게 따르면, 또 하나의 흔한 AI 대시보드가 아니라 Linear, Raycast, Stripe, GitHub 같은 감각에 가까운 UI를 만들 수 있습니다.

이것이 Uncodixify입니다.

## 평범하게 만들어라 (Uncodexy-UI 기준)

- 사이드바: 평범하게
  240~260px 고정 폭, 단색 배경, 단순한 오른쪽 보더, 바깥쪽 큰 둥근 모서리 없음
- 헤더: 평범하게
  단순한 텍스트만, eyebrow 없음, 대문자 라벨 없음, 그라디언트 텍스트 없음, 그냥 적절한 계층의 h1/h2
- 섹션: 평범하게
  20~30px 정도의 일반적인 패딩, 대시보드 안에서 히어로 블록 금지, 장식용 문구 금지
- 내비게이션: 평범하게
  단순한 링크, 은은한 hover, transform 애니메이션 없음, 기능적 목적이 아니면 배지 금지
- 버튼: 평범하게
  단색 배경 또는 단순 보더, 반경 8~10px 이하, 알약 모양 금지, 그라디언트 배경 금지
- 카드: 평범하게
  단순 컨테이너, 반경 8~12px 이하, 은은한 보더, 8px blur 이상의 그림자 금지, 떠 있는 느낌 금지
- 폼: 평범하게
  표준 입력 필드, 위쪽에 명확한 라벨, fancy floating label 금지, 단순한 포커스 상태
- 입력창: 평범하게
  단색 보더, 단순 포커스 링, 애니메이션 밑줄 금지, 형태 변형 금지
- 모달: 평범하게
  중앙 정렬, 단순한 배경 오버레이, 슬라이드 인 애니메이션 금지, 직관적인 닫기 버튼
- 드롭다운: 평범하게
  단순 리스트, 은은한 그림자, fancy 애니메이션 금지, 명확한 선택 상태
- 테이블: 평범하게
  깔끔한 행, 단순 보더, 은은한 hover, 필요 없으면 zebra stripe 금지, 좌측 정렬
- 리스트: 평범하게
  단순 아이템, 일관된 간격, 장식용 불릿 금지, 명확한 계층
- 탭: 평범하게
  단순 underline 또는 border indicator, pill 배경 금지, sliding 애니메이션 금지
- 배지: 평범하게
  작은 글씨, 단순 배경 또는 보더, 반경 6~8px, glow 금지, 필요할 때만 사용
- 아바타: 평범하게
  단순 원형 또는 라운드 사각형, 장식 보더 금지, 기능적 이유 없는 상태 링 금지
- 아이콘: 평범하게
  단순한 형태, 16~20px 일관 크기, 장식용 아이콘 배경 금지, 단색 또는 은은한 색
- 타이포그래피: 평범하게
  시스템 폰트 또는 단순 sans-serif, 명확한 계층, 억지 serif/sans 혼합 금지, 본문 14~16px 가독성
- 간격: 평범하게
  4/8/12/16/24/32px 일관된 스케일, 랜덤한 갭 금지, 과도한 패딩 금지
- 보더: 평범하게
  1px solid, 은은한 색, 두꺼운 장식용 보더 금지, 그라디언트 보더 금지
- 그림자: 평범하게
  최대 0 2px 8px rgba(0,0,0,0.1), 극적인 그림자 금지, 컬러 그림자 금지
- 트랜지션: 평범하게
  100~200ms ease, 바운스 금지, transform 효과 금지, 단순 opacity/color 변화
- 레이아웃: 평범하게
  표준 grid/flex, 창의적인 비대칭 금지, 예측 가능한 구조, 명확한 정보 위계
- 그리드: 평범하게
  일관된 컬럼, 표준 gap, 겹침 금지, 반응형 브레이크포인트 사용
- 플렉스박스: 평범하게
  단순 정렬, 표준 gap, 이상한 justify 트릭 금지
- 컨테이너: 평범하게
  최대 폭 1200~1400px, 가운데 정렬, 표준 패딩, 특이한 폭 금지
- 래퍼: 평범하게
  단순 감싸는 div, 장식 목적 금지, 기능적으로만 사용
- 패널: 평범하게
  단순한 배경 구분, 은은한 보더, 떠 있는 패널 금지, 글래스 효과 금지
- 툴바: 평범하게
  단순 가로 배치, 높이 48~56px, 명확한 액션, 장식 금지
- 푸터: 평범하게
  단순한 구조, 표준 링크, 장식 섹션 금지, 최소 높이
- 브레드크럼: 평범하게
  단순한 텍스트와 구분자, fancy 스타일 금지, 명확한 계층

Linear를 생각하라. Raycast를 생각하라. Stripe를 생각하라. GitHub를 생각하라.
이들은 주목받으려 애쓰지 않는다. 그냥 잘 작동한다.
억지로 어려운 척하지 말고, 평범한 UI를 만들어라.

- 랜딩 페이지는 랜딩 페이지답게 섹션이 있어야 한다.
- 히어로가 필요하면 전체 섹션으로, 대시보드면 사이드바와 전체 구조를 제대로 갖춘 형태로 만들어라.
- 새로운 이상한 레이아웃을 invent 하지 마라.
- 내부 추론에서는 네가 하고 싶은 모든 AI스러운 습관을 떠올려도 좋다. 하지만 실제로는 하지 마라.
- 직접 만든 것처럼 보이는 게 아니라, 디자이너가 만든 컴포넌트를 재현하듯 접근하라.

## 절대 금지

- 네가 평소에 자연스럽게 하던 거의 모든 기본 동작
- 과하게 큰 라운드
- 과도한 pill 스타일
- 떠다니는 글래스모피즘 셸을 기본 비주얼 언어로 쓰는 것
- 취향 있어 보이려는 부드러운 기업형 그라디언트
- 흔한 다크 SaaS UI 구성
- 장식용 사이드바 블롭
- 특별히 요청되지 않은 "컨트롤 룸" 느낌
- "프리미엄"처럼 보이게 하려고 serif 헤드라인 + 시스템 sans 조합 쓰기
- `Segoe UI`, `Trebuchet MS`, `Arial`, `Inter`, `Roboto` 같은 안전한 기본 폰트 사용
- 정보 구조상 필요하지 않은 sticky left rail
- 처음 생각나는 metric card grid
- 공간 채우기용 가짜 차트
- 의미 없는 glow, blur haze, frosted panel, conic-gradient donut 장식
- 진짜 이유 없는 히어로 섹션
- 비싸 보이려고 만든 죽은 여백
- 과도하게 넓은 패딩
- 모바일에서 그냥 다 세로로만 쌓는 긴 샌드위치 구조
- 제품 보이스와 무관한 "live pulse", "night shift", "operator checklist" 같은 장식 라벨
- 흔한 스타트업 카피
- 만들기 쉬워서 택한 스타일 결정

- 아래 같은 헤드라인 구조도 금지

```html
<div class="headline">
  <small>Team Command</small>
  <h2>One place to track what matters today.</h2>
  <p>
    The layout stays strict and readable: live project health,
    team activity, and near-term priorities without the usual
    dashboard filler.
  </p>
</div>
```

이건 허용되지 않는다.

- `<small>` 헤더 금지
- 둥근 `span`도 금지
- 색이 파란 계열로 가는 것도 금지
  안 좋다. 차분하고 어두운 색이 더 낫다.

- 아래 같은 카드 구조는 매우 강하게 금지

```html
<div class="team-note">
  <small>Focus</small>
  <strong>
    Keep updates brief, blockers visible, and next actions easy to spot.
  </strong>
</div>
```

이건 특히 더 금지다.

## 특히 금지되는 패턴들

- 20px~32px 반경을 모든 곳에 반복 적용하는 것
- 사이드바, 카드, 버튼, 패널에 같은 둥근 사각형 반복
- 280px 정도 폭의 사이드바와 상단 브랜드 블록 + 그 아래 링크 구조
- 떠 있는 분리형 사이드바
- 이유 없이 유리 카드 안에 들어간 canvas chart
- 도넛 차트와 애매한 퍼센트 조합
- hierarchy 대신 glow에 의존한 카드
- 왼쪽 정렬과 중간 부유형 정렬이 섞여 기준이 없는 레이아웃
- 지나치게 gray-blue로 빠진 대비 약한 텍스트
- 사실상 블루-블랙 그라디언트 + 시안 포인트인 "프리미엄 다크 모드"
- 템플릿처럼 느껴지는 타이포그래피
- eyebrow 라벨
- 대시보드 안의 full hero strip
- "Operational clarity without the clutter" 같은 장식성 카피
- 설명용 미니노트의 남발
- hover 시 translateX 같은 transform 애니메이션
- 과한 박스 그림자
- `::before`로 찍는 상태 점 장식
- 대문자 + letter-spacing 과용한 muted label
- 그라디언트 pipeline bar
- KPI 카드 3열 그리드
- "Team focus", "Recent activity" 류 패널
- 상태 태그를 남발한 테이블
- 사이드바 안 CTA가 들어간 workspace 블록
- 그라디언트 브랜드 마크
- 카운트나 "Live" 상태가 붙은 nav 배지
- quota/usage 패널과 progress bar
- 메타 정보 줄이 달린 푸터
- 색으로만 처리한 trend indicator
- 오른쪽 rail의 "Today" 일정 패널
- panel, panel-2, rail-panel, table-panel 등 여러 패널 유형 중첩

## 규칙

UI 선택이 "AI가 기본으로 고를 법한 것"처럼 느껴진다면 금지하고,
더 어렵더라도 더 깔끔한 선택을 해라.

- 색은 차분해야 한다. 싸우면 안 된다.

- 너는 색을 고르는 데 약하다. 아래 우선순위를 따르라.

1. 가장 우선: 사용자의 프로젝트에 이미 있는 색을 사용
2. 프로젝트에 팔레트가 없으면 아래 미리 정의된 팔레트 중 하나를 참고
3. 랜덤 색 조합은 만들지 마라

첫 번째 팔레트만 고를 필요는 없다. 적절한 것을 하나 선택해서 참고하면 된다.
